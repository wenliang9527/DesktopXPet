# 养成系统

## 概述

DesktopXPet 的养成系统让宠物具有生命力——属性会随时间变化，互动会影响宠物状态，工作会带来成长。

## 四维属性

| 属性 | 范围 | 衰减速率 | 恢复方式 |
|------|------|---------|---------|
| 心情 (mood) | 0-100 | 0.5/分钟 | 互动、喂食 |
| 饱食度 (satiety) | 0-100 | 0.5/分钟 | 喂食 |
| 精力 (energy) | 0-100 | 工作-0.3/分钟 | 睡眠+0.5/分钟 |
| 亲密度 (intimacy) | 0-100 | 不衰减 | 所有互动 |

**特殊规则：**
- 饱食度低于 20 时，心情额外衰减 1/分钟
- 精力为 0 时，宠物强制进入 idle 状态
- 所有属性 clamp 在 [0, 100]

## 互动方式

| 操作 | 动画 | 属性变化 | XP | 冷却 |
|------|------|---------|-----|------|
| 左键点击 | 跳跃 | 心情+3、亲密度+1 | +1 | 3秒 |
| 悬停1.5秒 | 摸头 | 心情+5、亲密度+2 | +1 | 3秒 |
| 右键喂食 | 吃饭 | 饱食度+20、心情+2、亲密度+1 | +3 | 3秒 |

每种互动有独立的冷却计时器，3秒内重复操作不会重复增加属性。

**喂食动画触发方式**：由主进程广播 `interact` 类型驱动，不再依赖 satiety 变化量推断。任何 `satiety < 100` 时的喂食都会触发吃饭动画（详见 [喂食动画触发](#喂食动画触发)）。

## 等级系统

### 经验值获取

| 行为 | XP |
|------|-----|
| 工作每分钟 | +1 |
| 任务完成事件 | +10 |
| 番茄钟完成 | +15 |
| 点击互动 | +1 |
| 喂食互动 | +3 |

### 升级

当 XP 累积到阈值时自动升级：
- 每级所需 XP = level × 100
- 升级后 XP 减去阈值，剩余保留
- 等级无上限

## 皮肤状态联动

养成系统与皮肤系统联动，宠物会根据属性值自动切换状态图。皮肤通过 `manifest.json` 的 `states` 字段声明支持的状态（详见皮肤文档）。

**状态决策优先级**（从高到低）：

1. 监控状态：`working` / `error` / `happy`
2. 养成状态：已解锁且触发条件满足的 nurture 状态
3. 基础兜底：`idle` / `sleeping` / `waking`

**养成状态分三类**（`category` 字段）：

| 类别 | 含义 | 示例状态 |
|------|------|---------|
| physiological（生理） | 反映生理需求 | `hungry`（饥饿）、`tired`（疲惫）、`sick`（生病） |
| emotion（情绪） | 反映情绪状态 | `sad`（难过）、`lonely`（孤独）、`excited`（兴奋） |
| behavior（行为） | 反映行为动作 | `daze`（发呆）、`walk`（散步）、`dance`（跳舞）等 |

**匹配规则**：
- `sleeping` 状态下只匹配 `physiological` 类别（如疲惫导致的睡眠）
- `idle` / `waking` 状态下匹配所有类别
- 多个状态同时满足触发条件时，取 `priority` 最高的（并列取第一个）
- 无 `states` 配置的皮肤（如默认橘猫）不受影响，只使用基础 6 状态

## 等级解锁

养成系统等级解锁两类内容：养成状态 和 特殊皮肤。

### 养成状态解锁

`manifest.states` 中每个状态可设 `unlockLevel`，等级不足时该状态不会触发（即使属性满足触发条件）。可在设置中启用/禁用等级解锁（禁用后所有状态视为已解锁）。

阈值优先级：`customThresholds`（用户自定义） > `manifest.states[].unlockLevel` > 默认解锁阈值表 > `1`。

**默认解锁阈值表**（状态未在 manifest 中声明 `unlockLevel` 时使用）：

| 状态 | 默认解锁等级 |
|------|------------|
| daze | Lv2 |
| hungry | Lv3 |
| tired | Lv3 |
| walk | Lv3 |
| angry | Lv4 |
| sad | Lv6 |
| lonely | Lv6 |
| excited | Lv6 |
| love | Lv8 |
| shy | Lv8 |
| sick | Lv9 |
| celebrating | Lv10 |
| dance | Lv10 |
| run | Lv10 |

### 皮肤解锁

皮肤 manifest 中设 `unlockLevel`，等级不足时皮肤完全隐藏（不会出现在皮肤选择器中）。

- 示例：婚纱皮肤 `unlockLevel=10`，Lv10 解锁后才出现在皮肤选择器中
- 升级时会广播皮肤列表变更，渲染进程自动重新拉取已解锁皮肤列表

## 状态切换冷却

每个养成状态有独立的切换冷却时间，防止属性在阈值边界波动时频繁切换状态图。

**冷却时间优先级**：`manifest.states[].cooldownMs`（状态专属） > 全局冷却时间（默认 `3000ms`）。

- 可在设置中调整全局冷却时间
- 仅 nurture 来源的状态切换会记录冷却（monitor / base 状态不占用冷却）
- 皮肤切换时清空冷却记录，避免新皮肤的状态被旧冷却阻塞
- 示例：satiety 在 20 边界波动时，`hungry` 状态不会在 `idle` / `hungry` 之间频繁闪烁

## 喂食动画触发

喂食动画通过主进程广播 `interact` 类型驱动，不依赖 satiety 变化量推断。

- **触发条件**：任何 `satiety < 100` 时的喂食都会触发吃饭动画
- **已满反馈**：`satiety` 已满（100）时显示"吃饱了~"气泡反馈，不触发动画
- **不误触发**：番茄钟完成（+10 satiety）不会误触发吃饭动画（因为不是 `interact` 操作）

> 旧实现从 `nurtureState.vitals.satiety` 变化量 `>= 15` 推断喂食，但 `clamp(satiety+20, 100)` 导致 `satiety>=86` 时实际增量 `< 15`，动画丢失。新实现由主进程 `interact()` 直接广播类型，不受 clamp 影响。

## 离线衰减

关闭应用后，属性会持续衰减：
- 基于实际经过时间计算（非定时器次数）
- 最多计算 2 小时（防止暴跌）
- 离线时按 idle 状态衰减（不会消耗精力）
- 重新打开应用时立即结算

## 番茄钟

### 工作循环
- 25 分钟工作 → 5 分钟休息 → 循环
- 每 4 个番茄钟后长休息 15 分钟

### 持久化
- 完成的番茄钟数量存储在养成系统中
- 重启应用后自动恢复
- 可在仪表盘中重置计数

## 数据安全

- 属性数据自动保存（5秒防抖）
- 应用退出时强制保存
- 存档损坏时自动回退到默认值
- 支持 schema 版本迁移

## 在仪表盘中查看

打开仪表盘（`Ctrl+Shift+D` 或双击宠物），可以看到：
- 等级和经验值进度条
- 四维属性实时数值
- 累计统计（完成任务数、工作总时长、番茄钟数）
- 当前显示状态（联动决策结果，含状态名、来源 monitor/nurture/base、触发原因）
- 已解锁状态列表（当前等级已解锁的 nurture 状态名）
