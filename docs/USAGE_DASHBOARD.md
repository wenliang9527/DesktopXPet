# 仪表盘与多任务监控

## 打开仪表盘

三种方式打开仪表盘：

1. **快捷键** — `Ctrl+Shift+D`
2. **右键宠物** → `📊 打开仪表盘`
3. **双击宠物**

仪表盘是一个独立的 800×600 窗口，不会影响宠物窗口。

## 仪表盘布局

### 1. 摘要卡片（顶部）

| 卡片 | 说明 |
|------|------|
| 宠物状态 | 当前宠物状态（idle/working/happy/error），带颜色 |
| 工作中 | 当前 working 状态的工具数量 |
| 出错 | 当前 error 状态的工具数量 |
| 总计工具 | 已接入的工具总数 |

### 2. 并行工作概览

当有工具处于 working 状态时显示：

```
┌──────────────────────────────────────────────────┐
│ ⚡ 并行工作中 (3)                                 │
│                                                    │
│  🚀 trae  Editing App.tsx    🧠 claude-code  Gen  │
│  🤖 Ollama  Running llama3                        │
└──────────────────────────────────────────────────┘
```

每个工具显示为标签（chip），包含图标、名称、摘要。

### 3. 状态时间线

显示最近 12 次宠物状态变化，用柱状条可视化：

- 柱高按状态映射：error 最高(100)、working(80)、happy(60)、idle(20)、sleeping(15)
- 颜色：working=绿色、error=红色、completed=蓝色、其他=灰色
- 鼠标悬停显示时间戳

### 4. 已接入工具（分组列表）

按来源分两组：

**内置插件组**：System、GitHub、Ollama（通过 Poll 轮询获取状态）

每行显示：
```
🖥️  System     CPU 45%, Memory 62%     [工作中]  刚刚
```

**外部工具组**：通过 HTTP Push 接入的工具（Trae、Claude Code、OpenCode 等）

每行显示：
```
🚀  trae       Editing App.tsx:42      [工作中]  3秒前
```

### 5. 实时状态详情

每个工具一张卡片，展示完整信息：

```
┌─────────────────────────────────┐
│ 🖥️ System              [working] │
│ CPU 85%, Memory 72%             │
│ cpu: 85                         │
│ memory: 72                      │
│ uptime: 3600                    │
│ platform: win32                 │
│ arch: x64                       │
└─────────────────────────────────┘
```

### 6. 番茄钟 + 皮肤选择器 + 设置面板

底部并排显示：
- **🍅 番茄钟** — 番茄工作法计时器(25分钟工作+5分钟休息循环)
  - 大字号倒计时显示,带阶段颜色(工作红/休息绿/待机灰)
  - 进度条可视化剩余时间
  - 开始/暂停/跳过/重置按钮
  - 可配置工作时长(1-120分钟)和休息时长(1-60分钟)
  - 阶段结束时系统通知 + 音效提醒
  - 显示已完成番茄数
- **宠物名称** — 可编辑的宠物名称，双击气泡或在此处点击"编辑"修改
- 皮肤选择器（网格预览，悬停动画预览，点击切换）
- 设置面板（宠物/行为/监控/通用设置）

## 多任务监控原理

### 数据来源

```
┌─────────────────────────────────────────┐
│           MonitorService                │
│                                         │
│  ┌───────────┐    ┌──────────────────┐ │
│  │ Poll 轮询  │    │ Push 推送缓存     │ │
│  │ (内置插件) │    │ (外部工具, 60s TTL)│ │
│  └─────┬─────┘    └────────┬─────────┘ │
│        │                   │           │
│        └───────┬───────────┘           │
│                ▼                       │
│         aggregateAll()                 │
│                │                       │
│                ▼                       │
│      AggregatedStatus                  │
│      { petState, tools[], summary }    │
└────────────────┬────────────────────────┘
                 │
         ┌───────┴───────┐
         ▼               ▼
   宠物窗口动画     仪表盘窗口
   状态气泡         工具列表
   悬停详情面板     时间线
```

### 状态聚合规则

多个工具的状态聚合为宠物状态：

| 条件 | 宠物状态 |
|------|----------|
| 任一工具 error | `error` |
| 任一工具 working | `working` |
| 30 秒内有工具 completed | `happy` |
| 以上都不满足 | `idle` |

### Push 数据有效期

外部工具通过 HTTP Push 推送的状态有 60 秒 TTL（Time To Live）：
- 60 秒内未收到新的 Push，该工具状态自动从聚合中移除
- 内置插件（Poll）的状态不受 TTL 影响

## 宠物窗口的多任务显示

### 状态气泡（StatusBubble）

当多个工具同时工作时，气泡显示：

```
┌──────────────────────────────┐
│ 3 个工具并行中                │
│ System: CPU 85%, Memory 72%  │
│ 🚀 trae: Editing App.tsx     │
│ 🤖 Ollama: Running llama3    │
│ +1 更多...                    │
└──────────────────────────────┘
```

- 最多显示 3 个 working 工具
- 超过 3 个显示 `+N 更多...`
- error 工具单独列出

### 悬停详情面板（StatusDetailPopup）

鼠标悬停在宠物上时显示完整工具列表：

```
┌──────────────────────────────┐
│ 📡 工具状态    3 工作中       │
│──────────────────────────────│
│ 🖥️ 🔄 System                 │
│      CPU 85%, Memory 72%     │
│ 🚀 🔄 trae                   │
│      Editing App.tsx:42      │
│ 🤖 🔄 Ollama                 │
│      Running llama3          │
│──────────────────────────────│
│ System: CPU 85% | trae: ...  │
└──────────────────────────────┘
```

从 appStore 实时读取，无需轮询。
